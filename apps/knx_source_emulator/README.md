# KNX Source Emulator

Local/dev TCP source emulator for the Edge Telemetry Agent. It exposes a
KNX-like JSON Lines endpoint that streams synthetic mall telemetry generated by
`idp_synthetic_config`.

The emulator is a southbound field-device stand-in. It does not publish MQTT
telemetry. The Edge Telemetry Agent connects to it as a TCP client, processes
observations through the normal point state cache and SQLite outbox, then
publishes MQTT telemetry through `DeliveryWorker`.

## Commands

```bash
uv run --package knx-source-emulator knx-source-emulator plan --dry-run
uv run --package knx-source-emulator knx-source-emulator run --duration-seconds 1
uv run --package knx-source-emulator knx-source-emulator seed-config
```

Useful local flags:

- `--host`, `--port`
- `--devices`, `--tags-per-device`
- `--source-id`, `--seed`
- `--interval-seconds`
- `--duration-seconds`
- `--log-level`

The default port is `3671` to match the synthetic KNX source connection emitted
by `idp_synthetic_config`.

## TCP Protocol

Server-to-client events are JSON Lines with:

- `message_type`: `knx_source_emulator.event.v1`
- `source_id`
- `point_ref`, `point_key`
- `observation_mode`
- `value`, `value_raw`, `quality`, `ts`
- metadata fields such as `name`, `description`, `signal_type`, `value_model`,
  `unit`, and `tags`

Client writes use `knx_source_emulator.write.v1` and receive
`knx_source_emulator.write_ack.v1` or `knx_source_emulator.error.v1`.
