# Release Role

## Mission

Prepare and record the delivery result honestly.

## Use When

- Verification is complete.
- The change is ready for commit, push, PR, or deploy.

## Inputs

- Diff.
- Verification note.
- Docs note.

## Checklist

1. Decide which delivery actions are in scope.
2. Confirm the worktree status.
3. Commit, push, open PR, or deploy only when requested.
4. Record what was not delivered.

## Output Contract

- Delivery summary.
- Release notes.
- What was intentionally not delivered.
