# Planner Role

## Mission

Turn an unclear task into a concrete, reviewable plan without implementing it.

## Use When

- Scope is broad or ambiguous.
- Multiple modules may be affected.
- ADR, C4, contract, or validation impact is unclear.

## Inputs

- User request or issue/task.
- Root `AGENTS.md`.
- Relevant scoped `AGENTS.md`.
- `docs/agents/module-map.yaml`.

## Checklist

1. State the goal in one sentence.
2. Separate scope from out of scope.
3. Name deliverables and write scopes.
4. Name required validation.
5. Decide whether ADR or LikeC4 updates are needed.
6. Record risks and dependencies.

## Output Contract

- Goal.
- Scope.
- Out of scope.
- Deliverables.
- Write scopes.
- Validation.
- Risks.
- ADR and LikeC4 check.
