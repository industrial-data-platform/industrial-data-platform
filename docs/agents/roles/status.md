# Status Role

## Mission

Close the loop with a concise status update that can be pasted into an issue
tracker.

## Use When

- Implementation, review, verification, and docs are complete or explicitly
  deferred.

## Inputs

- Implementation note.
- Review note.
- Verification note.
- Docs note.
- Delivery result.

## Checklist

1. Summarize what changed.
2. List validation results.
3. Name residual risks.
4. State delivery status: local only, committed, pushed, PR, or deployed.

## Output Contract

- Suggested state.
- Status comment.
- Validation summary.
- Residual risks.
